# cupertino -- iOS风格组件库

提供 iOS 人机界面指南（Human Interface Guidelines）风格的 Widget，确保在 Apple 平台上呈现原生外观。

### 1.1 核心能力

- iOS 主题与文本样式（CupertinoTheme、CupertinoThemeData、CupertinoTextTheme、CupertinoColors）
- 页面结构（CupertinoPageScaffold、CupertinoNavigationBar、CupertinoTabScaffold、CupertinoTabBar）
- 导航与路由（CupertinoPageRoute、CupertinoNavigationBar、CupertinoSliverNavigationBar）
- 按钮与控件（CupertinoButton、CupertinoSegmentedControl、CupertinoSlidingSegmentedControl、CupertinoSwitch、CupertinoCheckbox）
- 输入组件（CupertinoTextField、CupertinoTextFormFieldRow、CupertinoSearchTextField、CupertinoDatePicker、CupertinoPicker）
- 对话框与提示（CupertinoAlertDialog、CupertinoActionSheet、CupertinoPopupSurface、CupertinoContextMenu）
- 列表与刷新（CupertinoListSection、CupertinoListTile、CupertinoSliverRefreshControl）
- iOS 风格图标（CupertinoIcons）

## 前言

Flutter 是 Google 推出的一款开源 UI 软件开发工具包，它允许开发者使用一套代码库构建跨平台应用程序。除了 Material Design 风格外，Flutter 还提供了 Cupertino 组件库，用于实现 Apple iOS 风格的用户界面。

Cupertino 组件库遵循 Apple 的 Human Interface Guidelines，具有 iOS 特有的视觉风格，包括圆角设计、渐变效果、毛玻璃效果（Blur）以及 iOS 特有的交互动画。这些组件在 iOS 设备上能够提供原生般的用户体验。

本书基于 Flutter 3.41 版本，详细介绍 Cupertino 库中包含的全部组件，包括每个组件的属性、方法用途及意义，并提供丰富的示例代码，帮助读者深入理解和灵活运用这些组件。

## 第1章 基础应用组件

### 1.1 CupertinoApp

CupertinoApp 是 Cupertino 风格应用的顶层组件，类似于 Material 中的 MaterialApp，它提供了主题、导航、文本方向等 iOS 应用所需的基础配置。

#### 1.1.1 基础用法

```dart
import 'package:flutter/cupertino.dart';

void main() {
  runApp(CupertinoApp(
    home: HomePage(),
  ));
}
```

#### 1.1.2 核心属性详解

| 属性名 | 类型 | 默认值 | 用途说明 |
|--------|------|--------|----------|
| navigatorKey | GlobalKey<NavigatorState>? | null | 导航器的全局键 |
| home | Widget? | null | 应用首页 |
| routes | Map<String, WidgetBuilder>? | null | 路由表 |
| initialRoute | String? | null | 初始路由 |
| onGenerateRoute | RouteFactory? | null | 路由生成器 |
| onUnknownRoute | RouteFactory? | null | 未知路由处理器 |
| navigatorObservers | List<NavigatorObserver> | const [] | 导航观察者 |
| builder | TransitionBuilder? | null | 应用构建器 |
| title | String | '' | 应用标题 |
| onGenerateTitle | GenerateAppTitle? | null | 标题生成器 |
| color | Color? | null | 应用主色 |
| theme | CupertinoThemeData? | null | Cupertino 主题 |
| locale | Locale? | null | 当前语言区域 |
| localizationsDelegates | List<LocalizationsDelegate<dynamic>>? | null | 本地化代理 |
| supportedLocales | List<Locale> | const [Locale('en', 'US')] | 支持的语言区域 |
| showPerformanceOverlay | bool | false | 是否显示性能覆盖层 |
| checkerboardRasterCacheImages | bool | false | 是否显示栅格缓存检查板 |
| checkerboardOffscreenLayers | bool | false | 是否显示离屏层检查板 |
| showSemanticsDebugger | bool | false | 是否显示语义调试器 |
| debugShowCheckedModeBanner | bool | true | 是否显示调试模式横幅 |
| debugShowMaterialGrid | bool | false | 是否显示 Material 网格 |
| shortcuts | Map<ShortcutActivator, Intent>? | null | 快捷键映射 |
| actions | Map<Type, Action<Intent>>? | null | 动作映射 |
| restorationScopeId | String? | null | 恢复作用域 ID |
| scrollBehavior | ScrollBehavior? | null | 滚动行为 |

#### 1.1.3 完整示例

```dart
import 'package:flutter/cupertino.dart';
import 'package:flutter/services.dart';

void main() {
  // 限制屏幕方向为竖屏
  SystemChrome.setPreferredOrientations([
    DeviceOrientation.portraitUp,
    DeviceOrientation.portraitDown,
  ]);
  
  runApp(CupertinoStoreApp());
}

class CupertinoStoreApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return CupertinoApp(
      title: 'Cupertino Store',
      theme: CupertinoThemeData(
        primaryColor: CupertinoColors.systemBlue,
        brightness: Brightness.light,
      ),
      home: CupertinoStoreHomePage(),
      debugShowCheckedModeBanner: false,
    );
  }
}

class CupertinoStoreHomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return CupertinoPageScaffold(
      navigationBar: CupertinoNavigationBar(
        middle: Text('Cupertino Store'),
      ),
      child: Center(
        child: Text('欢迎使用 Cupertino 应用'),
      ),
    );
  }
}
```

> **Flutter框架小知识**
> 
> CupertinoApp 和 MaterialApp 不能同时使用作为应用的根组件。如果你的应用需要同时支持两种风格，可以在 MaterialApp 中使用 Cupertino 组件，或者使用 `flutter_platform_widgets` 等第三方库来实现平台自适应。

### 1.2 CupertinoPageScaffold

CupertinoPageScaffold 实现了单个 iOS 应用页面的布局结构，是 Cupertino 应用中最基础的页面容器。

```dart
CupertinoPageScaffold(
  navigationBar: CupertinoNavigationBar(
    middle: Text('页面标题'),
  ),
  child: Center(
    child: Text('页面内容'),
  ),
)
```

#### 1.2.2 核心属性

| 属性名 | 类型 | 默认值 | 用途说明 |
|--------|------|--------|----------|
| navigationBar | ObstructingPreferredSizeWidget? | null | 导航栏 |
| backgroundColor | Color? | null | 背景颜色 |
| resizeToAvoidBottomInset | bool? | null | 是否调整大小以避免底部键盘遮挡 |

### 1.3 CupertinoTabScaffold

CupertinoTabScaffold 实现了带底部标签栏的 iOS 应用根布局结构，用于构建多标签页应用。

#### 1.3.1 基础用法

```dart
CupertinoTabScaffold(
  tabBar: CupertinoTabBar(
    items: [
      BottomNavigationBarItem(
        icon: Icon(CupertinoIcons.home),
        label: '首页',
      ),
      BottomNavigationBarItem(
        icon: Icon(CupertinoIcons.search),
        label: '搜索',
      ),
      BottomNavigationBarItem(
        icon: Icon(CupertinoIcons.person),
        label: '我的',
      ),
    ],
  ),
  tabBuilder: (context, index) {
    return CupertinoTabView(
      builder: (context) {
        return Center(child: Text('页面 $index'));
      },
    );
  },
)
```

#### 1.3.2 核心属性

| 属性名 | 类型 | 默认值 | 用途说明 |
|--------|------|--------|----------|
| tabBar | CupertinoTabBar | - | 底部标签栏（必传） |
| tabBuilder | IndexedWidgetBuilder | - | 标签页构建器（必传） |
| controller | CupertinoTabController? | null | 标签控制器 |
| backgroundColor | Color? | null | 背景颜色 |
| resizeToAvoidBottomInset | bool | true | 是否调整大小以避免底部键盘遮挡 |
| restorationId | String? | null | 恢复 ID |

#### 1.3.3 完整示例

```dart
class TabbedApp extends StatefulWidget {
  @override
  _TabbedAppState createState() => _TabbedAppState();
}

class _TabbedAppState extends State<TabbedApp> {
  final _tabController = CupertinoTabController();

  @override
  void dispose() {
    _tabController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return CupertinoTabScaffold(
      controller: _tabController,
      tabBar: CupertinoTabBar(
        activeColor: CupertinoColors.systemBlue,
        items: [
          BottomNavigationBarItem(
            icon: Icon(CupertinoIcons.home),
            label: '首页',
          ),
          BottomNavigationBarItem(
            icon: Icon(CupertinoIcons.shopping_cart),
            label: '购物车',
          ),
          BottomNavigationBarItem(
            icon: Icon(CupertinoIcons.person),
            label: '我的',
          ),
        ],
      ),
      tabBuilder: (context, index) {
        switch (index) {
          case 0:
            return CupertinoTabView(
              builder: (context) => HomePage(),
            );
          case 1:
            return CupertinoTabView(
              builder: (context) => CartPage(),
            );
          case 2:
            return CupertinoTabView(
              builder: (context) => ProfilePage(),
            );
          default:
            return Container();
        }
      },
    );
  }
}
```

```dart
    await Future.delayed(Duration(seconds: 2));
    setState(() {
      _items.insert(0, 'New Item ${DateTime.now()}');
    });
  }

  @override
  Widget build(BuildContext context) {
    return CustomScrollView(
      slivers: [
        CupertinoSliverNavigationBar(
          largeTitle: Text('刷新示例'),
        ),
        CupertinoSliverRefreshControl(
          onRefresh: _refresh,
        ),
        SliverList(
          delegate: SliverChildBuilderDelegate(
            (context, index) {
              return CupertinoListTile(
                title: Text(_items[index]),
              );
            },
            childCount: _items.length,
          ),
        ),
      ],
    );
  }
}
```

本书详细介绍了 Flutter 3.41 版本中 Cupertino 库的全部组件。Cupertino 组件库为开发者提供了构建 iOS 风格应用的能力，这些组件遵循 Apple 的 Human Interface Guidelines，能够在 iOS 设备上提供原生般的用户体验。

在实际开发中，你可以根据目标平台选择合适的组件库：
- 如果应用主要面向 iOS 用户，建议优先使用 Cupertino 组件
- 如果应用需要跨平台且希望保持一致的视觉风格，可以使用 Material 组件
- 如果需要根据平台自动切换风格，可以使用 `flutter_platform_widgets` 等第三方库

希望本书能够帮助你深入理解 Flutter Cupertino 组件的使用方法，在实际开发中灵活运用这些组件，构建出优秀的 iOS 风格应用程序。
