# 变量

变量是程序中最基本的存储单元，用于存储数据。

## 第1章 变量声明

Dart 提供两种方式来声明变量：一种是使用关键字`var` 、`final` 、`const` ，另一种则是使用类型名称。

### 1.1 关键字

#### 1.1.1 `var` -- 类型推断

编译器会自动推断变量类型，但类型一旦确定就不可更改。

```dart
void main() {
  // var 声明的变量类型由初始化值确定
  var name = 'Alice';		// ✅：推断为 String
  
  name = 'Doe';		// ✅：允许
  
  // 类型确定后不能改变
  // name = 123;	// ❌：错误，不能将 int 赋值给 String
}
```

#### 1.1.2 `final` -- 不可变引用

`final` 变量只能赋值一次，赋值后不可修改。特性如下：

* 运行时常量：值在运行时确定；
* 实例变量可以是`final` ，但不能是`const` ；
* 引用不可变：修饰引用类型时，引用不可变，但对象内部状态可变；

```dart
void main() {
  // final 变量只能赋值一次
  final String name = 'Alice';
  
  name = 'Bob';   // ❌ 错误：不能重新赋值
  
  // 类型推断
  final age = 25;  // int
  
  // final 集合的内容可以修改
  final List<int> numbers = [1, 2, 3];
  numbers.add(4);  // ✅ 正确
  numbers = [5, 6];  // ❌ 错误：不能重新赋值
  
  // 运行时常量
  final currentTime = DateTime.now(); // 运行时才确定值
}
```

#### 1.1.3 `const` -- 编译时常量

编译时常量，值必须在编译时就确定。特性如下：

* 编译时常量：值在编译时计算并内联；
* 内存优化：相同的`const` 值在内存中只存在一份；
* 深度不可变：对象及其内部状态都不可修改；
* 隐式`final` ：所有`const` 变量都是隐式的`final` ； 

```dart
void main() {
  // const 变量必须在编译时确定
  const String appName = 'MyApp';
  const int maxCount = 100;
  const double pi = 3.14159;
  
  // const 集合也是深度不可变的
  const List<int> numbers = [1, 2, 3];
  numbers.add(4);  // ❌ 运行时错误：无法修改不可变列表
  
  // const 构造函数
  const point = Point(10, 20);
  
  // 不能用于运行时值
  const now = DateTime.now();  // ❌ 编译时错误
}

class Point {
  final int x, y;
  const Point(this.x, this.y);
}
```

### 1.2 类型名称

#### 1.2.1 显式类型声明

```dart
void main() {
  // 显式声明变量类型
  String name = 'Alice';
  int age = 25;
  double height = 1.75;
  bool isActive = true;
  List<String> hobbies = ['reading', 'coding'];
  Map<String, dynamic> profile = {'city': 'Beijing', 'score': 95.5};
}
```

#### 1.2.2 `dynamic` -- 动态类型

可以存储任何类型的值，且可以随时改变类型。

```dart
dynamic data = 'Hello';

data = 123;        // ✅ 编译通过，类型变为 int

data = [1, 2, 3];  // ✅ 编译通过，类型变为 List<int>
```

#### 1.2.3 `Object` 

所有类的基类，可以存储任何对象，但需要显示转换才能使用具体方法。

```dart
Object obj = 'Hello';

print(obj.length);   // ❌ 编译错误：Object 没有 length 属性

print((obj as String).length);  // ✅ 需要显示转换
```

###  1.3 常见问题

####  1.3.1 `var` vs 显示类型

```dart
// ✅ 推荐：使用 var，代码更简洁
var name = 'Alice';

// ✅ 也推荐：复杂类型显式声明，提高可读性
Map<String, List<int>> scores = {
  'math': [90, 85, 95],
};

// ❌ 不推荐：冗余的类型声明
String name2 = 'Bob';  // 类型显而易见
```

#### 1.3.2 `final` vs `const` 

| 特性         | `final`    | `const`            |
| :----------- | :--------- | :----------------- |
| 赋值时机     | 运行时     | 编译时             |
| 值确定时间   | 首次使用时 | 编译阶段           |
| 内存位置     | 堆         | 编译时常量池       |
| 集合内容修改 | ✅ 允许     | ❌ 不允许           |
| 重新赋值     | ❌ 不允许   | ❌ 不允许           |
| 实例变量     | ✅ 允许     | ❌ 不允许（仅静态） |

```dart
void main() {
  // 内存优化示例
  const a = [1, 2];
  const b = [1, 2];
  print(identical(a, b)); // true - 指向同一内存地址
}
```

#### 1.3.3 `dynamic` vs `Object` 

| 特性                 | `dynamic`                        | `Object`                                       |
| -------------------- | -------------------------------- | ---------------------------------------------- |
| 静态类型检查         | ❌ 关闭 - 编译器信任你的调用      | ✅ 开启 - 编译器会检查方法调用                  |
| 允许调用不存在的方法 | ✅ 编译通过，运行时可能报错       | ❌ 编译时报错                                   |
| 设计意图             | ”我不确定类型，让运行时来决定“   | ”可以是任何东西，但我只使用 Object 的基本能力“ |
| 空安全               | 等同于`Object?` ，但关闭类型检查 | `Object(非空)` 、`Object?(可空)`               |

```dart
/// Object 示例
void printObject(Object obj) {
  // 只能调用 Object 类定义的方法（toString、hasCode、runtimeType等）
  print(obj.toString());      // ✅ 编译通过

  // obj.nonExistentMethod(); // ❌ 编译错误：Object 没有这个方法
}

void main() {
  printObject('Hello'); // ✅
  printObject(123);     // ✅
  printObject(null);    // ❌ 编译错误：Object 不接受 null
}


/// dynamic 示例
void printDynamic(dynamic obj) {
  print(obj.toString());      // ✅ 编译通过

  // 编辑器完全信任你，即使方法不存在也能编译通过
  obj.nonExistentMethod();    // ✅ 编译通过，但运行时抛出 NoSuchMethodErr
  obj + 10;                   // ✅ 编译通过，但如果 obj 不是数字会报错
}

void main() {
  printDynamic('Hello');
  print(123);
  print(null);                // ✅ dynamic 接受 null
}
```

## 第2章 变量初始化

### 2.1 默认值

未初始化的变量默认值为`null` ，变量在使用前必须先初始化（使用`var` 、`dynamic`和`可空类型` 声明变量除外）。

```dart
void main() {
  // 1. 声明后立即初始化
	String name = 'Alice';
  
  // 2. 先声明，使用前再初始化
	int age;
	age = 28;
  
  // 3. 变量在使用前必须先初始化
  int count;
  print(count);  // ❌ 编译时错误：count 使用前必须先初始化

  // 4. var、dynamic、可空类型声明的变量使用前可以不初始化
  var name;
  print(name);  // ✅ 输出：null
}
```

### 2.2 `late` -- 延迟初始化

`late` 用于声明非空变量但延迟初始化，主要有以下用途：

```dart
// 1. 延迟初始化
late String description;

// 2. 懒加载：如果 temperature 从未被使用，readThermometer()永远不会执行
late String temperature = readThermometer();
```

#### 2.2.1 延迟初始化（late）

```dart
void main() {
  // late 变量：声明时不初始化
  late String description;

  // print(description); // ❌ 编译时错误：使用前必须赋值

  description = 'This is a description';
  print(description);
}
```

#### 2.2.2 延迟执行初始化（late + 函数）

```dart
void main() {
  // 延迟计算：只在首次访问时执行
  late final expensiveValue = calculateExpensiveValue();

  print('Before access');
  print(expensiveValue); // 此时才执行 calculateExpensiveValue
  print(expensiveValue); // 直接使用缓存值
}

int calculateExpensiveValue() {
  print('Calculating...');
  return 42;
}

// 依次输出：
// Before access
// Calculating...
// 42
// 42
```

#### 2.2.3 使用场景

##### 1. 与空安全配合

```dart
class User {
  // 非空字段，延迟初始化
  late String name;
  
  void init(String userName) {
    name = userName;
  }
}

void main() {
  final user = User();
  print(user.name);  // ❌ 运行时错误：name 未初始化

  user.init('Alice');
  print(user.name);  // ✅ Alice
}
```

##### 2. 性能优化（替换getter）

```dart
class ChartWdigetState {
  // 每次 build 都会重新计算
  ChartEntry get minChartEntry = widget.chartData.calculateMin();

  // 只计算一次，后续复用结果
  late final ChartEntry minChartEntry = widget.chartData.calculateMin();
}
```

## 第3章 空安全（Null Safety）

Dart 2.12+ 引入了空安全，类型默认不可为空。旨在帮助开发者在编译时发现潜在的空指针异常，减少运行时崩溃。

#### 1.2.1 可空类型

```dart
void main() {
  // 非空类型（默认）
  String name = 'Alice';
  // name = null;  // 错误：不能将 null 赋值给非空类型
  
  // 可空类型（类型后加 ?）
  String? nullableName = 'Alice';
  nullableName = null;  // ✅ 正确
  
  // 延迟初始化的非空变量
  late String lateName;
  lateName = 'Bob'; // 首次赋值
}
```

#### 1.2.2 空断言运算符（!）

```dart
void main() {
  String? nullableName = 'Alice';
  
  // 使用 ! 断言非空
  String name = nullableName!;  // 如果 nullableName 为 null，会抛出异常
  
  // 安全使用：先检查再断言
  if (nullableName != null) {
    String name2 = nullableName;  // 类型提升，无需 !
    print(name2.length);
  }
  
  // 或者
  String name3 = nullableName ?? 'Default Name';
}
```

#### 1.2.3 空感知运算符

```dart
void main() {
  String? name;
  
  // ?? 空合并运算符
  String displayName = name ?? 'Unknown';
  print(displayName);  // Unknown
  
  // ??= 空合并赋值运算符
  name ??= 'Default';
  print(name);  // Default
  
  // ?. 空感知访问
  print(name?.length);  // 7
  
  // ?.. 空感知级联
  StringBuffer? buffer;
  buffer?..write('Hello')..write(' World');
  
  // ?[] 空感知索引
  List<int>? numbers;
  print(numbers?[0]);  // null
}
```

## 第4章 变量作用域

### 4.1 词法作用域

Dart 使用词法作用域（静态作用域），变量的作用范围由代码结构决定。

```dart
// 顶级（全局）变量
String globalName = 'Global'; 

void main() {
  // 函数级变量
  String mainName = 'Main';  // main 函数局部变量
  
  void innerFunction() {
    String innerName = 'Inner';  // 内部函数局部变量
    
    print(globalName);  // ✅ 访问全局变量
    print(mainName);    // ✅ 访问外部变量
    print(innerName);   // ✅ 访问局部变量
  }
  
  innerFunction();
  // print(innerName);  // 错误：innerName 不在作用域内
}
```

### 4.2 变量遮蔽（Shadowing）

Dart 的作用域遵循大括号 `{}`  规则。

```dart
void main() {
  String name = 'Outer';
  
  {
    String name = 'Inner';  // 遮蔽外部 name
    print(name);  // Inner
  }
  
  print(name);  // Outer
}
```

### 4.3 私有变量

以下划线 `_` 开头的标识符是库私有的。

```dart
// 在 lib/utils.dart 中
String _privateVar = '库私有'; // 只在当前库可见
String publicVar = '公开';    // 所有库可见
```

### 4.3 闭包与变量捕获

```dart
void main() {
  var count = 0;
  
  // 闭包捕获 count 变量
  void increment() {
    count++;
    print('Count: $count');
  }
  
  increment();  // Count: 1
  increment();  // Count: 2
  increment();  // Count: 3
}
```

## 第5章 高级特性

### 5.1 类型提升

```dart
void main() {
  String? name = getName();
  
  // 类型提升：检查 null 后，Dart 知道 name 不为 null
  if (name != null) {
    print(name.length);  // 无需 ! 运算符
  }
  
  // 使用 ?? 后也会提升
  final displayName = name ?? 'Unknown';
  print(displayName.length);  // displayName 是非空 String
}

String? getName() => 'Alice';
```

### 5.2 延迟加载（懒加载）

```dart
class ExpensiveResource {
  // 使用工厂构造函数实现懒加载单例
  static ExpensiveResource? _instance;
  
  factory ExpensiveResource() {
    _instance ??= ExpensiveResource._internal();
    return _instance!;
  }
  
  ExpensiveResource._internal() {
    print('Creating expensive resource...');
  }
}

// 使用 late 实现懒加载
class Service {
  late final _database = _openDatabase();
  
  Database get database => _database;
  
  Database _openDatabase() {
    print('Opening database...');
    return Database();
  }
}

class Database {}
```

### 5.3 通配符变量（dart 3.7+）

使用 `_` 作为非绑定变量名，可多次声明而不冲突：

```dart
var (x, _) = (1, 2);
var (y, _) = (3, 4); // 不会冲突

// 函数参数
void foo(int _, String _) {
  // 两个 _ 是不同的变量，互不影响
}
```

### 5.4 数字分割符（dart 3.6+）

使用下划线 `_` 提高数字可读性：

```DART
var million = 1_000_000;
var binary = 0x0000_0000_0000_0000;
var decimal = 1__000_000__000_000__000_000; // 多个下划线也允许
```

## 第6章 最佳实践

### 6.1 命名规范

```dart
// ✅ 变量名使用小驼峰
var userName = 'Alice';
var itemCount = 10;

// ✅ 私有变量以下划线开头
var _internalCounter = 0;

// ✅ 常量使用小驼峰或全大写下划线
const maxCount = 100;
const DEFAULT_TIMEOUT = Duration(seconds: 30);

// ❌ 避免单字母变量（循环除外）
var n = 'Alice';  // 不好

// ✅ 使用有意义的名称
var userDisplayName = 'Alice';  // 好
```

### 6.2 变量选择指南

```dart
/// 变量类型选择决策树
/// 
/// 1. 值是否永远不会改变？
///    ├── 编译时已知 → const
///    └── 运行时确定 → final
///
/// 2. 值会改变吗？
///    ├── 类型明显 → var
///    └── 类型复杂 → 显式类型
///
/// 3. 需要延迟初始化？
///    └── 使用 late

class VariableGuide {
  // ✅ 编译时常量
  static const appName = 'MyApp';
  static const version = '1.0.0';
  
  // ✅ 运行时常量
  final String id = generateId();
  final DateTime createdAt = DateTime.now();
  
  // ✅ 可变变量
  var clickCount = 0;
  var userStatus = 'active';
  
  // ✅ 延迟初始化
  late final Database database;
  
  void init() {
    database = Database();
  }
}

String generateId() => 'ID_${DateTime.now().millisecondsSinceEpoch}';
class Database {}
```

### 6.3 常见陷阱

```dart
void commonPitfalls() {
  // 陷阱 1：late 变量未初始化就使用
  late String name;
  // print(name);  // 运行时错误
  
  // 陷阱 2：final 集合内容可修改
  final List<int> numbers = [1, 2, 3];
  numbers.add(4);  // ✅ 可以修改内容
  // numbers = [5, 6];  // ❌ 不能重新赋值
  
  // 陷阱 3：const 集合深度不可变
  const List<int> constNumbers = [1, 2, 3];
  // constNumbers.add(4);  // ❌ 运行时错误
  
  // 陷阱 4：dynamic 失去类型安全
  dynamic value = 'Hello';
  value = 123;
  // value.substring(0, 3);  // 运行时错误
  
  // 陷阱 5：空安全下的类型转换
  String? nullable;
  // String nonNull = nullable;  // 编译错误
  String nonNull = nullable!;  // 运行时可能错误
}
```
