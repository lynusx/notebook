# 模块系统

导入（import）和导出（export）是 Dart 模块系统的核心，用于组织代码、管理依赖和控制命名空间。

## 第1章 `import`

### 1.1 基本导入

```dart
// 导入 Dart 核心库
import 'dart:io';
import 'dart:async';
import 'dart:convert';

// 导入第三方包
import 'package:http/http.dart';
import 'package:path/path.dart';

// 导入本地文件
import 'utils.dart';
import 'models/user.dart';
import '../common/helpers.dart';
```

### 1.2 URI 类型

```dart
// 1. dart: URI - Dart SDK 库
import 'dart:math';
import 'dart:typed_data';

// 2. package: URI - 依赖包
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

// 3. 相对路径 - 本地文件
import './models.dart';      // 同级目录
import '../utils.dart';      // 上级目录
import 'src/internal.dart';  // 子目录

// 4. 绝对路径（不推荐）
// import '/home/user/project/lib/utils.dart';
```

### 1.3 导入路径的推荐写法

```dart
// ✅ 推荐：使用相对路径导入本地文件
import 'models/user.dart';
import '../utils/helpers.dart';

// ✅ 推荐：使用 package: 导入包内其他文件
import 'package:my_app/models/user.dart';

// ❌ 不推荐：混合使用相对和绝对路径
import 'models/user.dart';
import 'package:my_app/models/order.dart';  // 同一包内应统一
```

### 1.3 部分导入（part/part of）

```dart
// ===== main_library.dart =====
library my_library;

import 'dart:async';

// 包含其他文件
part 'src/models.dart';
part 'src/utils.dart';
part 'src/extensions.dart';

class MainClass {
  void usePartClass() {
    final model = PartModel();  // 来自 part 文件
    final result = partUtil();   // 来自 part 文件
  }
}

// ===== src/models.dart =====
part of my_library;

class PartModel {
  String name = '';
}

// ===== src/utils.dart =====
part of my_library;

String partUtil() => 'utility';
```

**Flutter 框架小知识**

**part 与 import 的区别**

| 特性 | part | import |
|------|------|--------|
| 命名空间 | 共享同一命名空间 | 独立的命名空间 |
| 访问权限 | 可访问私有成员 | 只能访问公共成员 |
| 使用场景 | 大文件拆分 | 模块依赖 |
| 代码生成 | 常用于代码生成 | 一般依赖 |

```dart
// 代码生成中的典型用法（如 freezed、json_serializable）
// person.dart
part 'person.freezed.dart';
part 'person.g.dart';

@freezed
class Person with _$Person {
  factory Person({required String name, required int age}) = _Person;
  factory Person.fromJson(Map<String, dynamic> json) => _$PersonFromJson(json);
}
```

## 第2章 `export` 

### 2.1 基本导出

```dart
// ===== widgets.dart =====
// 将多个文件统一导出
export 'buttons/primary_button.dart';
export 'buttons/secondary_button.dart';
export 'cards/info_card.dart';
export 'cards/product_card.dart';
export 'inputs/text_field.dart';

// 使用者只需导入一个文件
// import 'package:my_app/widgets.dart';
```

### 2.2 选择性导出

```dart
// ===== api.dart =====
// 只导出公共 API，隐藏内部实现
export 'src/client.dart' show ApiClient, RequestOptions;
export 'src/models.dart' show User, Order, Product;
export 'src/exceptions.dart' show ApiException, NetworkException;

// 隐藏特定成员
export 'src/internal.dart'
  show PublicClass
  hide _PrivateClass, internalHelper;
```

### 2.3 导出重命名

```dart
// ===== collections.dart =====
// 重命名导出的成员以避免冲突
export 'legacy_list.dart' hide List;
export 'new_list.dart';

// 或者使用前缀导出
export 'legacy_list.dart' as legacy;
export 'new_list.dart';
```

### 2.4 库的组织模式

```dart
// ===== my_package.dart =====
/// My Package - 一个示例 Dart 包
///
/// 提供用户管理和订单处理功能
library my_package;

// 核心功能
export 'src/user/user_service.dart';
export 'src/user/user_repository.dart';
export 'src/order/order_service.dart';
export 'src/order/order_repository.dart';

// 模型
export 'src/models/user.dart';
export 'src/models/order.dart';
export 'src/models/product.dart';

// 异常
export 'src/exceptions/api_exception.dart';
export 'src/exceptions/validation_exception.dart';

// 工具
export 'src/utils/validators.dart';
export 'src/utils/formatters.dart';

// 内部实现不导出
// src/internal/* 保持私有
```

## 第3章 库前缀与冲突解决

### 3.1 使用前缀（as）

```dart
// 避免命名冲突
import 'package:path/path.dart' as p;
import 'package:path_provider/path_provider.dart' as provider;

void pathExample() async {
  // 使用前缀区分不同库的功能
  final localPath = p.join('documents', 'file.txt');
  final appDir = await provider.getApplicationDocumentsDirectory();
}
```

### 3.2 选择性导入（show/hide）

```dart
// 只导入需要的成员
import 'dart:math' show pi, sin, cos, max, min;

void mathExample() {
  print(pi);           // ✅ 可用
  print(sin(0));       // ✅ 可用
  // print(tan(0));    // ❌ 未导入
}

// 隐藏不需要的成员
import 'dart:io'
  hide Platform  // 使用 package:platform 替代
  hide stdin, stdout, stderr;  // 不需要控制台 I/O

// 组合使用
import 'large_library.dart'
  show PublicClass, publicFunction
  hide _PrivateClass;
```

### 3.3 解决命名冲突

```dart
// 场景：两个库都有 Logger 类
import 'package:logging/logging.dart' as logging;
import 'package:logger/logger.dart' as logger_lib;

class MyApp {
  final logging.Logger _fileLogger = logging.Logger('FileLogger');
  final logger_lib.Logger _consoleLogger = logger_lib.Logger();
}

// 另一种方式：选择性导入
import 'package:logging/logging.dart' show Logger as FileLogger;
import 'package:logger/logger.dart' show Logger as ConsoleLogger;
```

### 3.4 扩展方法冲突

```dart
// ===== string_extensions.dart =====
extension StringExtensions on String {
  String capitalize() => 
    '${this[0].toUpperCase()}${substring(1)}';
}

// ===== another_extensions.dart =====
extension AnotherStringExtensions on String {
  String capitalize() => toUpperCase();  // 不同的实现
}

// 1.导入时解决冲突
import 'string_extensions.dart';
import 'another_extensions.dart' hide AnotherStringExtensions;

void example() {
  print('hello'.capitalize()); 	// 使用 string_extensions.dart 的实现
}

// 2.使用库前缀
import 'string_extensions.dart' as ext1;
import 'another_extensions.dart' as ext2;

void example() {
  print(ext1.StringExtensions('hello').capitalize());		// 使用ext1
  print(ext2.AnotherStringExtensions('hello').capitalize()); 	// 使用ext2
}
```

## 第4章 条件导入与延迟加载

### 4.1 条件导入（平台特定代码）

```dart
// ===== platform_helper.dart =====
// 根据平台导入不同实现
export 'platform_helper_stub.dart'
  if (dart.library.io) 'platform_helper_io.dart'
  if (dart.library.html) 'platform_helper_web.dart';

// ===== platform_helper_io.dart =====
import 'dart:io';

class PlatformHelper {
  static String get platform => Platform.operatingSystem;
  static bool get isMobile => Platform.isIOS || Platform.isAndroid;
  static String get userAgent => 'Dart/${Platform.version}';
}

// ===== platform_helper_web.dart =====
import 'dart:html';

class PlatformHelper {
  static String get platform => 'web';
  static bool get isMobile => window.navigator.userAgent.contains('Mobile');
  static String get userAgent => window.navigator.userAgent;
}

// ===== platform_helper_stub.dart =====
class PlatformHelper {
  static String get platform => throw UnsupportedError('Unsupported platform');
  static bool get isMobile => throw UnsupportedError('Unsupported platform');
  static String get userAgent => throw UnsupportedError('Unsupported platform');
}

// ===== 使用 =====
import 'platform_helper.dart';

void main() {
  print('Platform: ${PlatformHelper.platform}');
  print('Is Mobile: ${PlatformHelper.isMobile}');
}
```

**条件导入的条件：**

| 条件 | 说明 |
|------|------|
| `dart.library.io` | 原生平台（VM）可用 |
| `dart.library.html` | Web 平台可用 |
| `dart.library.js` | JavaScript 互操作可用 |
| `dart.library.ffi` | FFI 可用 |
| `dart.library.isolate` | Isolate 可用 |

### 4.2 延迟加载（deferred）

```dart
// ===== main.dart =====
import 'package:heavy_feature/heavy_feature.dart' deferred as heavy;

void main() async {
  // 应用启动时 heavy 库未加载
  print('App started');
  
  // 按需加载
  await heavy.loadLibrary();
  print('Heavy feature loaded');
  
  // 现在可以使用 heavy 库
  heavy.initializeFeature();
  heavy.processData();
}

// 带加载状态的处理
Future<void> useHeavyFeature() async {
  try {
    await heavy.loadLibrary();
  } catch (e) {
    print('Failed to load feature: $e');
    return;
  }
  
  heavy.doSomething();
}
```

**Flutter 框架小知识**

**延迟加载在 Flutter Web 中的应用**

```dart
// 将大型功能模块延迟加载，减少初始包大小
import 'package:chart_library/chart_library.dart' deferred as charts;

class ChartPage extends StatefulWidget {
  @override
  _ChartPageState createState() => _ChartPageState();
}

class _ChartPageState extends State<ChartPage> {
  bool _isLibraryLoaded = false;
  
  @override
  void initState() {
    super.initState();
    _loadLibrary();
  }
  
  Future<void> _loadLibrary() async {
    await charts.loadLibrary();
    setState(() => _isLibraryLoaded = true);
  }
  
  @override
  Widget build(BuildContext context) {
    if (!_isLibraryLoaded) {
      return CircularProgressIndicator();
    }
    return charts.ChartWidget();
  }
}
```

// ===== network_client.dart =====
export 'network_client_stub.dart'
  if (dart.library.io) 'network_client_io.dart'
  if (dart.library.html) 'network_client_web.dart';

// ===== network_client_io.dart =====
import 'dart:io';

class NetworkClient {
  final HttpClient _client = HttpClient();
  
  Future<String> get(String url) async {

    final response = await request.close();
    return await response.transform(utf8.decoder).join();
  }
}

// ===== network_client_web.dart =====

  }

## 第5章 库分片

### 5.1 `library` -- 库声明

`library` 用于显示声明一个库，它是 Dart 代码组织的顶层单元。虽然 Dart 中的每个文件默认就是一个库，但是显式使用`library` 声明可以带来命名、文档生成和元数据注解等优势。

#### 5.1.1 基本声明

```dart
// 简单声明
library my_library;
  
// 带文档注释
/// 这是一个用户管理库，提供用户认证和数据操作功能。
/// 
/// 使用示例
/// ```dart
/// final userManager = UserManager();
/// await userManager.login('user@example.com', 'password');
/// ```
library user_management;
```

#### 5.1.2 命名规范

```dart
// ✅ 推荐：小写 + 下划线
library user_management;
library data_repository;
library flutter_app_utils;

// ❌ 避免：驼峰命名或大写
library UserManagement;		// ❌ 不推荐
library dataRepository;		// ❌ 不推荐
```

#### 5.1.3 默认可选

对于单文件库，`library` 声明是**可选的**。如果不需要文档、注释或分片，通常省略。

```dart
// utils.dart
library utils; // 可选，通常省略

String capitalize(String s) => s[0].toUpperCase() + s.substring(1);
```

#### 5.1.4 库注解

* 配合`@Deprecated` 标记废弃库

```dart
// 整个库被标记为废弃，使用时会产生警告

@Deprecated('Use new_user_management instead. Will be removed in v3.0.0')
library user_management;
```

* 配合代码生成工具

```dart
// freezed 代码生成

// 1. library 指令必须放在最前面
@freezed
library user_models;

// 2. 然后是 import 指令
import 'package:freezed_annotation/freezed_annotation.dart';

// 3. 最后是 part 指令
part 'user_models.freezed.dart';
part 'user_models.g.dart';

// 所有在此定义的类都会生成 freezed 代码
```

### 5.2 `part` / `part of` 

`part` 和 `part of` 是 Dart 的库分片机制，用于将一个库拆分成多个物理文件，同时保持逻辑上的统一。

#### 5.2.1 `part of` -- 声明归属的库

在分片文件中使用，声明自己属于哪个主库：

```dart
// src/addition.dart
part of calculator;  // 声明：我是 calculator 的一部分

// 这里的代码逻辑上属于 calculator
int addImpl(int a, int b) => a + b;

// src/subtraction.dart
part of calculator;  // 声明：我是 calculator 的一部分

// 这里的代码逻辑上属于 calculator
int subtractImpl(int a, int b) => a - b;
```

#### 5.2.2 `part` -- 主库引入分片

在主库文件中使用，引入分片：

```dart
// calculator.dart (主库文件)
library calculator;  // 库声明（可选，但推荐）

part 'src/addition.dart';  			// 引入分片 addition.dart
part 'src/subtraction.dart';		// 引入分片 subtraction.dart

// 主库的公共 api
class Calculator {
  // 可以直接调用 part 文件中定义的函数
  int add(int a, int b) => addImpl(a, b);
  int subtract(int a, int b) => subtractImpl(a, b);
}
```

#### 5.2.3 `part` vs `import` 

| 指令     | `part` 、`part of`   | `import`             |
| -------- | -------------------- | -------------------- |
| 关系     | 同一库的不同文件     | 完全独立的库         |
| 命名空间 | 共享，无隔离         | 独立，需通过前缀访问 |
| 私有成员 | 互相可见             | 不可见（下划线成员） |
| 使用场景 | 拆分大文件，代码生成 | 依赖其他包/模块      |
| 编译结果 | 合并为一个编译单元   | 保持独立             |

### 5.3 常见误区

#### 5.3.1 每个文件都写`library` 

```dart
// ❌ 过度使用：简单工具文件不需要
library utils; // 多余，除非需要 part 或注释

import 'dart:math';

double calculate() => ...

// ✅ 推荐：直接开始写代码
import 'dart:math';

double calculate() => ...
```

#### 5.3.2 库名与文件名不一致

```dart
// file: user_manager.dart
library account_service; // ❌ 不一致，造成混淆

// ✅ 应该与文件名或 package 名保持一致
library user_manager;
// 或
library my_package.user_manager;
```

#### 5.3.3 在 `part` 文件中重复声明 `library` 

```dart
// main.dart
library my_lib;
part 'a.dart';

// a.dart
library other other_lib;
part of my_lib;
```

### 6.1 库设计原则

```dart
// ===== 好的库设计 =====

// 1. 清晰的公共 API

export 'src/models.dart' show User, Order, Product;
export 'src/services.dart' show UserService, OrderService;
export 'src/exceptions.dart' show ApiException;

// 2. 隐藏实现细节

// 3. 一致的命名
// 文件名使用小写下划线
// user_service.dart ✓
// userService.dart ✗
// UserService.dart ✗

// 4. 文档注释
/// A service for managing users.
///
/// Example:
/// ```dart
/// final service = UserService();
/// final user = await service.getUser('123');
/// ```
class UserService {
  /// Fetches a user by their ID.
  ///
  /// Throws [ApiException] if the user is not found.
  Future<User> getUser(String id) async {
    // ...
  }
}
```

### 6.2 导入排序规范

// 推荐的导入顺序：

// 陷阱 1：循环导入

```dart

bool isLibraryLoaded(String libraryName) {
  // 实际实现需要跟踪加载状态

/// 安全导入，处理导入失败
Future<T?> safeImport<T>(Future<T> Function() importer) async {
  try {
    return await importer();
  } catch (e) {
    print('Import failed: $e');
    return null;
  }
}

/// 条件导入辅助函数
T conditionalImport<T>({
  required T Function() io,
  required T Function() web,
  required T Function() otherwise,
}) {
  if (const bool.fromEnvironment('dart.library.io')) {
    return io();
  } else if (const bool.fromEnvironment('dart.library.html')) {
    return web();
  }
  return otherwise();
}
